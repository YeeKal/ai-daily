 ### 第一部分：系统工程视角的优化建议（解决重复推送）
 
 在提示词优化之前，关于“即时推送（Immediate Push）”的去重，**不建议完全依赖单个 Prompt 解决**。你需要给系统增加一点“记忆”。
 
 **建议流程：**
 1. **获取最新条目** -> **评分 (Score)**
 2. 如果分数 ≥ 90，提取该条目的 `summary`。
 3. **新增一步：查重判断 (Deduplication Check)**：将该条目与过去 24 小时内已推送的 `immediate_push` 列表进行对比。可以借助一个轻量级的 LLM 调用（或者向量检索）：*“新消息A与已推送的历史消息列表B，是否在报道同一个核心事件？返回 YES 或 NO。”*
 4. 如果是 YES，丢弃（或追加到晚间汇总）；如果是 NO，才进入 `immediate_push.txt` 生成快讯文案。

 ### 第二部分：提示词（Prompt）全面优化
 
 下面是为你重写的三个核心 Prompt。
 
 
 #### 2. 优化 `immediate_push.txt`（专业快讯风格，引入上下文防重）
 **优化思路：** 将其定位为彭博社/The Rundown AI 风格的快讯记者。传入最近 24 小时的推送记录，要求 LLM 发现事件的关联性，不仅陈述事实，还要给出“为什么重要（Why it matters）”。

 ```text
 ## 任务要求：
 1. **查重与关联**：如果原始数据中的某件事，在【过去 24 小时已推送历史】中已经出现过，**请忽略该条目**，不要重复推送。除非它提供了极其重大的**新进展**（例如：之前推送了谈判，现在推送了彻底破裂）。
 2. **专业聚合**：如果输入数据中有多条讨论**同一个事件**（例如一条是官方发布，另一条是行业大佬的深刻评价），请将它们**融合为一条快讯**，不要分开写。
 3. **专业视角**：不要只是简单翻译，要像资深编辑一样提炼出“核心事实（What）”以及“它的行业影响（Why it matters）”。
 4. **语气风格**：客观、犀利、克制。多用短句，重点内容加粗。
 ```

 #### 3. 优化 `digest.txt`（小互日报/AppSo 风格的每日精华整合）
 **优化思路：** 这是最关键的 Prompt。必须强制 LLM **打破原有的“一条推文一条新闻”的结构**，按照“大事件（Event）”进行融合，合并观点，输出结构化的日报。
 
 ```text
 --- START OF FILE digest.txt ---
 你是一位顶级的 AI 行业观察家与日报主编（风格参考小互日报、AppSo）。你的任务是将今天杂乱无章、来源各异的信息碎片，熔炼、重组成一篇**结构极度清晰、主次分明、洞察深刻**的《AI 资讯早报》。
 
 【待处理的碎片化信息】（共 {count} 条）：
 {entries}
 
 【近3天行业背景】（供洞察参考）：
 {context}
 
 ## 核心排版与整合规则（必须严格遵守）：
 1. **事件级融合（重中之重）**：绝不能把讨论同一件事的多条信息分开列出！例如，关于“Anthropic拒绝DoD”事件，可能包含了官方声明、Gary Marcus的分析、Sam Altman的内涵推文。你必须把它们**合并成一条新闻**，提炼全貌。
 2. **精选原则**：只挑选真正有价值的 5-8 个核心事件。丢弃纯情绪发泄、闲聊和过度重复的内容。
 3. **结构化呈现**：每个事件采用 “一句核心亮点 + 展开的无序列表” 结构，让人一目了然。
 4. **客观专业**：用简练的中文，避免低级词汇，像分析师一样指出事件的行业意义。保留所有相关的原文链接，附在每个事件末尾。
 ```
